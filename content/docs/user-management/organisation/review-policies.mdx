# Review Organisation Policies

Organisation policies are broad controls that apply before more specific work is evaluated. They are useful for rules that should be consistent across the workspace, such as IP rules or rate-limit boundaries.

For safety and guardrail concepts, see [Guardrails](/docs/security-and-guardrails/guardrails) and [Security Engine](/docs/security-and-guardrails/safetysec-engine).

<Steps>

<Step>

Open your organisation workspace.

</Step>

<Step>

In the sidebar, click **Settings**.

</Step>

<Step>

Find the organisation policies card.

</Step>

<Step>

Review whether the policy belongs at organisation level or should be narrower.

</Step>

<Step>

Use organisation policies for controls that should apply broadly across the workspace.

</Step>

<Step>

Use team or API key policies when the rule belongs to a smaller ownership boundary.

</Step>

</Steps>
