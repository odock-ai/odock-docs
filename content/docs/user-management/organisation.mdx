# Organisation

An organisation is the main tenant boundary. Providers, provider keys, models, teams, users, virtual API keys, budgets, quotas, MCP servers, and usage records are scoped to an organisation.

## Organisation Information

Organisation information includes name, contact, status, branding, and policies. Update these in organisation settings or from the super-admin organisation detail page.

## Organisation Routing

Organisation routing is a global switch. It only enables or disables smart routing for the organisation. The actual routing policy is configured per virtual API key. See [Routing](/docs/management/routing).

## Organisation Guardrail Policies

Organisation policies currently store IP rules and rate-limit settings under `policies`. These are resolved by the gateway with global, organisation, team, API key, and model/MCP scopes.

For request and response safety controls, see [Guardrails](/docs/security-and-guardrails/guardrails) and [SafetySec Engine](/docs/security-and-guardrails/safetysec-engine).
