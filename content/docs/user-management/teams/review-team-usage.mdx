# Review Team Usage

Team usage records help explain spend, errors, latency, model use, provider use, and whether access still matches real work.

For deeper usage reporting, see [Usage Monitoring](/docs/observability/usage-monitoring).

<Steps>

<Step>

Open the team detail page.

</Step>

<Step>

Go to **Usage Records**.

</Step>

<Step>

Review recent requests by time, model, provider, API key, status, and cost.

</Step>

<Step>

If a pattern needs a limit, review the team's **Budgets** or **Quotas**.

</Step>

<Step>

If a request should not have been possible, review the team API key's model and MCP access grants.

</Step>

</Steps>
