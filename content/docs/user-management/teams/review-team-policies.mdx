# Review Team Policies

Team policies are useful for controls that should apply to a group of workflows. Use API key policies when the rule belongs to one specific credential.

For broader safety concepts, see [Guardrails](/docs/security-and-guardrails/guardrails) and [SafetySec Engine](/docs/security-and-guardrails/safetysec-engine).

<Steps>

<Step>

Open **Teams**.

</Step>

<Step>

Click **Open** on the target team.

</Step>

<Step>

Find the team policies card.

</Step>

<Step>

Review whether the rule belongs to the team or to a specific API key.

</Step>

<Step>

Use team policies for controls shared by the team's workflows.

</Step>

</Steps>
