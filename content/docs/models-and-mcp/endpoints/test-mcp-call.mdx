# Test an MCP call

<Steps>

<Step>
Create or open a virtual API key.
</Step>

<Step>
Grant MCP Access for the MCP server.
</Step>

<Step>
Confirm the MCP server is enabled.
</Step>

<Step>
Confirm transport and auth settings are valid.
</Step>

<Step>
Send a request to `/v1/mcp/{slug}` with the Odock virtual API key.
</Step>

<Step>
Review the MCP server's **MCP Usage** table or the organisation **Usage Records** views.

> Screenshot placeholder: MCP server detail page showing API Key Access and MCP Usage after a test request.
</Step>

</Steps>
