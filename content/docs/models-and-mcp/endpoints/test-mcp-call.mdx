# Test an MCP call

Before testing, confirm the server transport, auth, and security model are configured. See [Streamable HTTP transport](/docs/models-and-mcp/mcp-servers/streamable-http-transport), [SSE transport](/docs/models-and-mcp/mcp-servers/sse-transport), [STDIO transport](/docs/models-and-mcp/mcp-servers/stdio-transport), [MCP authentication](/docs/models-and-mcp/mcp-servers/authentication), and [MCP security](/docs/models-and-mcp/mcp-servers/security).

<Steps>

<Step>
Create or open a virtual API key.
</Step>

<Step>
Grant MCP Access for the MCP server.
</Step>

<Step>
Confirm the MCP server is enabled.
</Step>

<Step>
Confirm transport and auth settings are valid.
</Step>

<Step>
Send a request to `/v1/mcp/{slug}` with the Odock virtual API key.
</Step>

<Step>
Review the MCP server's **MCP Usage** table or the organisation **Usage Records** views.

> Screenshot placeholder: MCP server detail page showing API Key Access and MCP Usage after a test request.
</Step>

</Steps>
