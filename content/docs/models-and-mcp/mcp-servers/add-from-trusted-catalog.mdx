# Add from the trusted catalog

Use the trusted catalog when the tool server is listed.

The catalog provides a starting point. Before production use, still review [MCP authentication](/docs/models-and-mcp/mcp-servers/authentication), [MCP security](/docs/models-and-mcp/mcp-servers/security), and [MCP pricing](/docs/models-and-mcp/mcp-servers/pricing).

<Steps>

<Step>
Open your organisation in Odock.
</Step>

<Step>
Go to **MCP Servers**.
</Step>

<Step>
Click the catalog or batch create action.
</Step>

<Step>
In **Select**, search by name, slug, description, or tag.
</Step>

<Step>
Filter by existing status, transport type, auth type, or category.

![Filter MCP Servers](/images/mcp-catalog-filters.png)
</Step>

<Step>
Select one or more new MCP servers.
</Step>

<Step>
Click through to **Configure**.
</Step>

<Step>
Optionally set Team Scope or API Key Scope.
</Step>

<Step>
For each selected server, review name, description, transport, endpoint or STDIO settings, auth type, auth config, enabled status, public internet flag, allowed tools, blocked tools, and metadata.

Use the transport pages when reviewing server behavior: [Streamable HTTP transport](/docs/models-and-mcp/mcp-servers/streamable-http-transport), [SSE transport](/docs/models-and-mcp/mcp-servers/sse-transport), and [STDIO transport](/docs/models-and-mcp/mcp-servers/stdio-transport).

![MCP config](/images/mcp-catalog-config.png)
</Step>

<Step>
Review the final step.
</Step>

<Step>
Create the selected MCP servers.

</Step>

</Steps>
