# Add from the trusted catalog

Use the trusted catalog when the tool server is listed.

<Steps>

<Step>
Open your organisation in Odock.
</Step>

<Step>
Go to **MCP Servers**.
</Step>

<Step>
Click the catalog or batch create action.
</Step>

<Step>
In **Select**, search by name, slug, description, or tag.
</Step>

<Step>
Filter by existing status, transport type, auth type, or category.

> Screenshot placeholder: Trusted MCP catalog dialog on the Select step with search, existing filter, transport filter, auth filter, category filter, grid/list toggle, server cards, tags, and official links.
</Step>

<Step>
Select one or more new MCP servers.
</Step>

<Step>
Click through to **Configure**.
</Step>

<Step>
Optionally set Team Scope or API Key Scope.
</Step>

<Step>
For each selected server, review name, description, transport, endpoint or STDIO settings, auth type, auth config, enabled status, public internet flag, allowed tools, blocked tools, and metadata.

> Screenshot placeholder: Trusted MCP catalog dialog on the Configure step showing Team Scope, API Key Scope, selected server transport settings, auth settings, allowed tools, blocked tools, and metadata.
</Step>

<Step>
Review the final step.
</Step>

<Step>
Create the selected MCP servers.

> GIF placeholder: Importing a GitHub-like MCP server from the trusted catalog, configuring auth and allowed tools, reviewing, and creating it.
</Step>

</Steps>
