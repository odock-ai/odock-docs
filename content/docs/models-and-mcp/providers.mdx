# Providers

Providers define upstream AI provider configuration: type, name, base URL, timeout, enabled flag, and optional JSON config.

## Provider Types

Supported provider types in the UI schema are `OPENAI`, `ANTHROPIC`, `GOOGLE`, `AZURE_OPENAI`, `VLLM`, and `CUSTOM`. Runtime provider implementations exist for OpenAI-compatible, Anthropic, Gemini, and vLLM paths.

## Add a provider

Create a provider from the organisation Providers page. Choose the type, enter a base URL, set a timeout if needed, and keep the provider enabled when it should be usable.

### Add Provider Key

Provider keys are upstream secrets used by the gateway when calling the actual provider. Add at least one provider key before creating usable models for that provider.

#### Provider Key Encryption

Provider keys are encrypted in the browser before they are stored. The UI sends an encrypted envelope and masked preview. `odock-server` decrypts the envelope only in memory when an upstream request needs the key.

Required alignment:

- UI build-time public variables: `NEXT_PUBLIC_PROVIDER_KEY_ENCRYPTION_KEY_ID`, `NEXT_PUBLIC_PROVIDER_KEY_ENCRYPTION_PUBLIC_KEY`
- Server private variables: `PROVIDER_KEY_ENCRYPTION_KEY_ID`, `PROVIDER_KEY_ENCRYPTION_PRIVATE_KEY`

The key IDs must match.

### Add Provider Models

After a provider and provider key exist, add models manually or from the model catalog. Each model must point to the provider and usually a provider key.
