# MCP Servers

MCP servers expose tools through the gateway. Odock governs them like models: they belong to an organisation, can be scoped to a team or API key, require explicit API key access, and produce usage records.

## MCP Servers Types

Transport types are `STREAMABLE_HTTP`, `SSE`, and `STDIO`. Auth types are `NONE`, `BEARER`, `BASIC`, and `OAUTH2`.

## Add Manual MCP Server

Add a manual MCP server when you know its endpoint or stdio command.

### Basic Informations & Slug

The `slug` is used in gateway calls: `/v1/mcp/{slug}`. Slugs are unique per organisation.

### Transport

Choose the transport based on how the MCP server runs.

#### Streamable HTTP

Use `STREAMABLE_HTTP` for HTTP MCP servers. Configure `endpointUrl`. Odock forwards JSON-RPC traffic to that URL.

#### SSE

Use `SSE` when the upstream MCP server streams events. Odock sets `Accept: text/event-stream` and records the request as streaming.

#### STDIO

Use `STDIO` when Odock should launch a local command such as `npx -y @modelcontextprotocol/server-sequential-thinking`. Configure command, args, and environment JSON.

### Auth, Access, Scope

Configure upstream auth with `NONE`, `BEARER`, `BASIC`, or `OAUTH2`. Then grant the virtual API key access through `ApiKeyMcpAccess`. Without that grant, the gateway returns `403 mcp_not_allowed`.

MCP server scope can also be narrowed to a team or specific API key.

### MCP Governance

Use allowed tools, blocked tools, semantic filters, policies, pricing, and access grants to govern MCP traffic. Allowed and blocked tools apply to `tools/call`; semantic filters can block configured keywords in the JSON-RPC payload.

## Add MCP Server from Catalog

The trusted MCP catalog loads from `NEXT_PUBLIC_TRUSTED_MCP_SERVERS_URL` or `/trusted-mcp-servers.json`.

### Select from Catalog

Select one or more trusted MCP servers, review the metadata, and import them into the organisation.

### Configure

After import, configure auth, access grants, policies, pricing, and tool allow/block rules before production use.

## MCP Pricing

MCP pricing supports call cost and input/output byte cost in nanos USD. Usage records include input bytes, output bytes, transport, RPC method, tool name, and total MCP cost.

## MCP Guardrails Policies

MCP policies include IP rules, rate limits, allowed tools, blocked tools, and semantic filters. See [Guardrails](/docs/security-and-guardrails/guardrails) for broader safety concepts.

## MCP API key Access

Grant access in the API key detail screen. MCP calls require both organisation match and an `ApiKeyMcpAccess` row.

## MCP Usage Records

MCP usage records capture server id, slug, transport, method, tool name, latency, input/output bytes, status, pricing snapshot, and total cost.
