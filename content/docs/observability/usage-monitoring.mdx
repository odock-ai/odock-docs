# Usage Monitoring

Usage monitoring is stored in Postgres and shown in the UI. It is the audit trail for what applications did through Odock.

## Usage Analytics

Usage analytics aggregate requests, tokens, errors, cost, latency, model, provider, organisation, team, user, and API key dimensions.

## Usage Record Table

The usage table lists individual requests. Use filters to inspect a model, provider, API key, team, user, budget, quota, or time window.

## Usage Record Details

Details include normalized token breakdown, billing, provider usage payload, routing attempts, latency, HTTP status, provider request id, and MCP artifacts when applicable.
