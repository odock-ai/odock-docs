# Tutorials

These tutorials use the organisation UI. They avoid admin-only screens and focus on the controls a user needs to configure and verify runtime guardrails.

## Before You Start

You need:

- an organisation in Odock
- at least one virtual API key
- a model or MCP server the key can call
- permission to edit the relevant organisation resources

For API key setup, see [Create an API key](/docs/management/virtual-api-keys/create-api-key). For model setup, see [Models](/docs/models-and-mcp/models). For MCP setup, see [MCP Servers](/docs/models-and-mcp/mcp-servers).
