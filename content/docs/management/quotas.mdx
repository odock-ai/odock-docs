# Quotas

Quotas enforce metric ceilings such as requests, tokens, errors, cost, and latency.

## Architecture and Owner Precedence

Quotas use the same owner order as budgets:

1. Organisation quotas.
2. Team quotas for team-scoped keys.
3. User quotas for user-scoped keys.
4. API key quotas.

If a reservation exceeds a quota, the gateway returns `429 quota_exceeded`.

## Add Quotas

Create quotas from the Quotas page. Pick owner type, metric, limit, period, schedule, and active state.

### Schedule, Periodicity & rollover

Periods are `DAILY`, `WEEKLY`, `MONTHLY`, and `QUARTERLY`. Use schedule fields to decide when limits start, stop, and reset.

### Quota Metric Value & Thresholds

Quota metrics are `REQUESTS`, `TOKENS`, `TOKENS_IN`, `TOKENS_OUT`, `ERRORS`, `COST`, and `LATENCY_MS`. Set the numeric limit for the selected metric.

## Quota Calendar

The quota calendar is the derived sequence of active quota windows.

## Quota Consumption Windows

Quota windows track used and reserved values per metric and period.

## Usage Records

Usage records provide request-level evidence for quota consumption, including status, token counts, latency, and cost.
