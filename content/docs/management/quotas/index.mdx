# Quotas

Quotas enforce numeric usage ceilings. Where budgets control spend, quotas control measurable runtime activity such as request count, token count, errors, cost, and latency.

Use quotas to protect workloads from unexpected volume, runaway jobs, broken integrations, and noisy clients.

## Quota Runtime Flow

```mermaid
flowchart TD
  REQ["Gateway request"]
  MATCH["Find active matching quotas"]
  ESTIMATE["Estimate metric usage"]
  RESERVE{"All quotas have capacity?"}
  DENY["429 quota_exceeded"]
  CALL["Provider or MCP call"]
  USAGE["Actual usage"]
  SETTLE["Settle reservation"]
  WINDOW["Quota window<br/>used + reserved"]

  REQ --> MATCH --> ESTIMATE --> RESERVE
  RESERVE -- "yes" --> CALL --> USAGE --> SETTLE --> WINDOW
  RESERVE -- "no" --> DENY
```
