# Budgets

Budgets enforce cost ceilings. Odock reserves estimated cost before an upstream call and settles actual cost after completion.

## Architecture and Owner Precedence

Budgets are collected in this order:

1. Organisation budgets.
2. Team budgets for team-scoped keys.
3. User budgets for user-scoped keys.
4. API key budgets.

All active matching budgets must have capacity. If a reservation exceeds a budget, the gateway returns `402 budget_exceeded`.

## Add budget

Create budgets from the Budgets page. Pick owner type, owner, period, currency, amount, thresholds, and active state.

### Schedule, Periodicity & rollover

Periods are `DAILY`, `WEEKLY`, `MONTHLY`, and `QUARTERLY`. Use `startAt`, `endAt`, timezone, and rollover to control budget windows.

### Budget Value & Thresholds

Budget amount is stored as nanos USD. In the UI, enter the human USD amount. Thresholds are percentages used for warning or alert workflows.

## Budget Calendar

The budget calendar is the schedule of active windows derived from the period, start date, end date, timezone, and rollover settings.

## Budget Consumption Windows

Budget windows store used and reserved nanos USD for each budget period. Reservations prevent concurrent requests from overspending a budget before actual usage is known.

## Usage Records

Use usage records and budget reservations together to reconcile cost. Usage records contain the final request cost; budget windows show consumption over time.
