# Create a budget

Use this workflow for organisation, team, user, or API-key spend control.

<Steps>

<Step>
Open your organisation workspace.
</Step>

<Step>
Open **Budgets** from the sidebar.
</Step>

<Step>
Click **Add New Budget**.
</Step>

<Step>
Enter a budget name.
</Step>

<Step>
Choose **Owner Type**: **Organisation**, **Team**, **User**, or **API Key**.
</Step>

<Step>
Select the team, user, or API key when the owner type requires it.
</Step>

<Step>
In **Schedule**, set start date, timezone, period, rollover, and optional end date.
</Step>

<Step>
In **Limits**, choose currency and enter **Amount (USD)**.
</Step>

<Step>
Select threshold percentages if you want warning points.
</Step>

<Step>
Keep **Active** checked if the budget should enforce immediately.
</Step>

<Step>
Create the budget.
</Step>

</Steps>

![Create a monthly budget for organisation](/gifs/budgets-create.gif)

In this example, we created a monthly budget for the entire organisation that ends in September.

## Add a weekly organisation budget

You can also define a weekly budget to avoid overspending before the monthly organisation budget is exhausted.

<Steps>

<Step>
Open **Budgets**.
</Step>

<Step>
Click **Add New Budget**.
</Step>

<Step>
Name the budget `Global weekly budget`.
</Step>

<Step>
Set **Owner Type** to **Organisation**.
</Step>

<Step>
Choose the start date. For a clean weekly schedule, pick the Monday when this budget should begin.
</Step>

<Step>
Set **Period** to **Weekly**.
</Step>

<Step>
Do not set an end date if the weekly budget should keep applying every week.
</Step>

<Step>
Enter the weekly amount and keep **Active** checked.
</Step>

<Step>
Create the budget.
</Step>

</Steps>

![Create a weekly budget for organisation](/images/budgets-weekly.png)


<Callout type="warning">
Each request in the organisation must comply with both the monthly and weekly budgets.
</Callout>

<Callout type="tip">
If the weekly limit is reached, requests are blocked even if funds remain in the monthly budget. The same applies in reverse: if the monthly budget is exhausted, requests are blocked even when the weekly budget still has remaining funds.
</Callout>
