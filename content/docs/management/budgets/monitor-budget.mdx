# Monitor a budget

Budget monitoring connects window-level consumption with request-level usage records.

<Steps>

<Step>
Open **Budgets**.
</Step>

<Step>
Filter by owner type, period, currency, or active state if needed.
</Step>

<Step>
Open the budget.
</Step>

<Step>
Review the **Budget consumption** card.

![Review a budget window](/gifs/budgets-windows.gif)

</Step>

<Step>
Use the previous and next controls to inspect older or newer windows.
</Step>

<Step>
Compare **Used**, **Limit**, and **Remaining**.
</Step>

<Step>
Scroll to **Usage Records** to inspect requests counted against the budget.
</Step>

<Step>
Open a usage record to inspect cost, model, provider, tokens, status, latency, and routing.
</Step>

</Steps>

![Review a budget](/gifs/budgets-overview.gif)

## What To Check

| Question | Where to look |
| --- | --- |
| Which window is active? | Budget consumption card and schedule calendar. |
| Is there in-flight usage? | Reserved value in the window. |
| What actually spent the budget? | Usage Records section. |
| Was a routed model involved? | Usage record Routing card. |
